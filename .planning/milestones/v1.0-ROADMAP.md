# 里程碑 v1.0: MVP

**状态:** ✅ 已发布 2026-05-12
**阶段:** 1-6
**计划总数:** 37

## 概述

完成的数据治理平台 MVP，提供基于 Go 的资产执行引擎、字段级血缘、Schema 演化追踪、带有 RBAC 的治理工作流，以及 React SPA + ConnectRPC API。

**前置依赖**: 无
**计划数**: 5 个计划

计划:
- [x] 01-01-PLAN.md: Go 模块引导, ent schema, Atlas 迁移, PostgreSQL 设置
- [x] 01-02-PLAN.md: 存储接口, 事件写入器, 追加式日志, RFC 7807 错误
- [x] 01-03-PLAN.md: JWT 服务, 认证中间件, 会话管理
- [x] 01-04-PLAN.md: 通过 hashicorp/go-plugin 实现稳定版本化的连接器接口
- [x] 01-05-PLAN.md: REST API (chi 路由器), 认证处理器, health/ready/metrics 端点

**详情:** 阶段 1 交付了平台基础: 通过 ent 实现 PostgreSQL 存储, Atlas 迁移, 带 RLS 保护的追加式事件日志, JWT 认证 (含邀请流程), 以及稳定的 v1.0.0 连接器 ABI。

## 里程碑总结

**关键决策:**
- 资产中心模型 (vs 任务中心) — Dagster 验证过的思维模型
- 纯 Go 核心, 无 Python 运行时 — 运维简洁
- 字段级血缘作为一等公民 — 与 Dagster 的主要差异化
- hashicorp/go-plugin 用于连接器可扩展性 — 进程隔离, 语言无关

**推迟的问题:**
- 阶段 6 质量趋势/告警存根 (ent 查询未连接) — INT-05 QualityEvaluator 连接缺口
- 阶段 6 AdminService 未实现 (ColumnPolicy schema 缺失) — INT-01 GovernanceWorkflow 连接缺口
- main.go 连接缺口 (GovernanceWorkflow, Enforcer, AuthMW 未连接) — 引导依赖注入不完整
- 阶段 4 EXPLAIN ANALYZE 捕获 (需要实时 PostgreSQL) — 人工 UAT 推迟
- 阶段 5 Snowflake/BigQuery/webhook 实时系统测试 — 需要真实凭证

**技术债务:**
- QualityEvaluator 未连接到执行器 (INT-05) — 质量规则从未评估
- GovernanceWorkflow 未连接到 API 依赖 (INT-01) — 治理 UI 功能不可用
- Casbin Enforcer 未连接 (INT-02) — RBAC 检查始终开放失败
- GovernanceGatingEnabled 未设置 (INT-04) — 执行器门是装饰性的
