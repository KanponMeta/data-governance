# Phase 2 Plan 03: Retry engine + global concurrency token pool + connector config + run executor - Summary

End-to-end run executor wiring retry, concurrency, configuration, and the per-run heartbeat goroutine — integrating plans 02-01 and 02-02 into a complete execution engine.

## What Was Built

### Task 1: concurrency_tokens table + Pool + retry/policy + retry_scheduled event

**ConcurrencyToken ent entity** (`internal/storage/ent/schema/concurrency_token.go`):
- Fields: id, run_id, asset_name, resource_tag, weight, acquired_at
- Indexes on resource_tag (capacity check), run_id (cleanup), acquired_at (stale detection)
- Single table for ALL concurrency limits (D-16 non-negotiable): run-level, op-level, resource-level

**Migration** (`migrations/20260507121500_phase2_concurrency_tokens.sql`):
- Hand-crafted SQL (atlas-provider-ent not available in network-isolated environment)
- Platform_owner ownership + platform_app DML grants matching Phase 1 pattern

**Pool** (`internal/concurrency/pool.go`):
- `Acquire(ctx, runID, assetName, tag, weight)`: BEGIN → SELECT SUM(weight) FOR UPDATE → check limit → INSERT → COMMIT. Returns ErrCapacity if limit would be exceeded.
- `Release(ctx, runID, tag)`: DELETE WHERE run_id=$1 AND resource_tag=$2. Idempotent.
- `ReleaseAll(ctx, runID)`: DELETE all tokens for a run. Called on terminal state.
- `ReleaseStale(ctx, staleAfter)`: DELETE WHERE acquired_at < cutoff. Called at worker startup (plan 02-04).
- Unconfigured tags get limit = 2^31-1 (unlimited) but still record the token row for observability.

**Retry policy** (`internal/retry/policy.go`):
- `Schedule(attempt, policy)`: `delay = InitialDelay * 2^(attempt-1)`, capped at MaxDelay, then jittered via `math/rand/v2`
- `ShouldRetry(attempt, policy)`: returns `attempt < policy.Max`; zero policy always returns false
- Attempt is 1-indexed: attempt=1 uses InitialDelay, attempt=2 uses 2×InitialDelay

**run.step.retry_scheduled event** (D-18 complete):
- `EventTypeRunStepRetryScheduled = "run.step.retry_scheduled"` added to AllKnownTypes()
- `RunStepRetryScheduledPayload{AssetName, Attempt, NextAttempt, ScheduledAt, DelayMs, Error}`
- Written to event_log BEFORE the delay sleep — audit trail persists even if worker crashes

### Task 2: Connector config loader + Run executor + heartbeat goroutine

**Config loader** (`internal/connector/config/config.go`):
- `Load(yamlBytes []byte) (*Config, error)`: resolves ${ENV_VAR} in raw bytes BEFORE yaml.Unmarshal
- `resolveEnv()`: `regexp.MustCompile(`\$\{([A-Z_][A-Z0-9_]*)\}`)` pattern, returns missing var names (never values)
- Secret values NEVER logged — Test 4 (TestLoad_DoesNotLogSecrets) asserts via slog capture
- Returns `ErrMissingEnvVar` wrapping all missing variable names on any unresolved placeholder
- `LoadFile(path)` reads from disk; PLATFORM_CONFIG env var convention documented

**FactoryRegistry** (`internal/connector/config/resolver.go`):
- `Factory func(params map[string]interface{}) (connector.Connector, error)` — plan 02-04 implements postgres factory
- `BuildAll(cfg *Config, reg *connector.Registry)` — iterates connectors, calls factories, calls RegisterInProcess
- Unknown connector types return formatted error; registration failures propagate

**Executor** (`internal/runtime/executor.go`):

```
Executor.Run(ctx, runID, assetName):
  ┌─ spawn heartbeatLoop goroutine (immediate first tick)
  ├─ Registry.Get(assetName) → build transitive subgraph → TopologicalOrder()
  ├─ transition starting → running → emit run.started
  ├─ for each asset in topological order:
  │   ├─ acquire global token + per-resource tokens (single pool, D-16)
  │   ├─ emit run.step.started
  │   ├─ safeMaterialize() [defer/recover wraps MaterializeFunc]
  │   ├─ release tokens
  │   ├─ on success: emit run.step.succeeded → continue
  │   └─ on failure: emit run.step.failed
  │       ├─ if ShouldRetry: scheduleRetry() → emit retry_scheduled → sleep → retry
  │       └─ else: fail run (emit run.failed, ReleaseAll, return error)
  └─ on all steps ok: transition running → succeeded → emit run.succeeded → ReleaseAll
  defer: hbCancel() + hbWG.Wait() [goroutine MUST exit before Run returns]
```

**heartbeatLoop** goroutine:
- Ticks `runs.last_heartbeat = NOW()` every HeartbeatInterval (default 30s)
- First tick is immediate so even very short runs leave a heartbeat trail
- Exits when hbCtx is canceled (deferred in Run's defer block)
- sync.WaitGroup ensures goroutine fully exits before Run returns (T-02-03-10)
- 10x safety margin: 30s tick vs 5m reaper threshold

## Two-Layer Fault Model (D-14 Option B)

| Fault Type | Handler | Mechanism |
|---|---|---|
| Business fault (MaterializeFunc returns error) | Engine retry loop | retry.Schedule + retry.ShouldRetry; inline re-execution within same Executor.Run |
| Infrastructure fault (worker crash, OOM, missed heartbeat) | Plan 02-04 stale-run reaper | Scans `last_heartbeat < NOW() - 5m`, calls TransitionForReset → queued |

River.max_attempts wiring explicitly DEFERRED to a future phase per D-14 Option B. The reaper (plan 02-04) is the canonical crash-recovery mechanism for Phase 2.

**2. [Rule 2 - Missing Critical] retry_scheduled event written before delay sleep**
- **Found during:** Task 1, Step H (event ordering analysis)
- **Issue:** Plan's scheduleRetry pattern wrote the event AFTER sleeping. If a worker crashed during the sleep, no retry_scheduled record would exist in event_log — violating T-02-03-06.

- **Files modified:** internal/runtime/executor.go
- **Commit:** 1ed9ac3
