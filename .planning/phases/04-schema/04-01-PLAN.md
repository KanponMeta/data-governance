import (
    "context"

// SeedDAG inserts a chain of asset_edges in PostgreSQL such that asset_0 → asset_1 → ... → asset_depth.
// Returns the asset names in topological order so callers can pick start nodes.

// SeedBranching inserts a balanced binary tree at the given depth (2^depth-1 assets, 2^depth-2 edges).
// Used to verify CTE handles fan-out, not just chains.

// SeedCycle inserts assets A→B→A so cycle-guard tests can verify the path-array NOT-contains check.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01-01 | Tampering | Schema A/B fixture pairs | low | accept | Test-only data, never executed against production. No runtime exposure. |
| T-04-01-02 | DoS | testcontainers container leak in CI | medium | mitigate | `t.Cleanup(func() { container.Terminate(...) })` in `StartPhase4Container` ensures teardown on test failure or panic. Smoke test verifies Cleanup hook runs. |
| T-04-01-03 | Information Disclosure | DB connection strings logged in test output | low | accept | testcontainers writes random credentials per run; no shared/persistent secrets. CI logs are internal-only. |
