<objective>
Land the two writers that turn Wave 1's empty tables into a useful lineage + schema audit trail:

1. **Lineage writer** (`internal/lineage/capture.go`):
   - `SyncStaticEdges(ctx, asset, codeHash)` — called from registry hook; idempotent UPSERT of asset_edges for every declared upstream (D-01 static derivation; D-15 soft-retire pattern).
   - `CaptureRun(ctx, tx, runID, asset, result, codeHash)` — runtime hook from executor; writes column_edges from runtime override OR builder default, emits `lineage.captured` event, detects observed-vs-declared upstream drift (D-04) and emits `lineage.drift_detected` + sets `asset_versions.drift_status='pending'`.
   - Both operations use the executor's transaction so a failed write rolls back the run state (D-21 crash recovery).

2. **Schema writer** (`internal/schema/capture.go` + `hash.go`):
   - `Capture(ctx, tx, runID, asset, result, conn, ref, codeHash)` — type-asserts `connector.SchemaDescriber`, calls `DescribeSchema` (or falls back to `result.Schema`, or tags `schema_capture_unsupported`), computes `schema_hash`, dedups against the latest `schema_versions` row for the asset (D-08), inserts a new row when changed and emits `schema.change_detected`, updates `last_seen_*` when unchanged and emits `schema.unchanged`, emits `schema.captured` for every successful capture, emits `schema.capture_failed` on any error (non-fatal — run continues).
   - Diff classification + writing schema_changes rows happens in **Wave 5 plan 04-05** (this plan's `schema.change_detected` event payload references the new `schema_versions.id`; Wave 5 reads that and writes the diff rows).

3. **Executor integration** (`internal/runtime/executor.go`):
   - Replace the current `e.transition(ctx, runID, run.StateRunning, run.StateSucceeded)` call (which uses raw ExecContext outside any tx) with a single transactional unit:
     ```
     tx, _ := db.BeginTx(...)
     lineageWriter.CaptureRun(tx, ...)
     schemaWriter.Capture(tx, ...)
     UPDATE runs SET state='succeeded' (in tx)
     tx.Commit()
     ```
   - On error: `tx.Rollback()` — run stays in `running`, heartbeat continues, reaper re-queues.

This plan does NOT introduce REST endpoints, CLI commands, recursive CTE traversals, or diff classification. Those are Waves 5–6.

**Drift detection logic (D-04 — PLATFORM-DRIVEN, NOT OPT-IN):**

D-04 specifies drift detection as a platform-driven invariant: every successful run is checked, no user opt-in. To implement this without changing the user-facing `AssetIO` interface (Phase 2 D-04 frozen contract), this plan introduces a **tracking decorator** in `internal/asset/io_tracking.go`:

```go
// trackingIO wraps an AssetIO and records every Read(upstream) call during materialize.
// The executor builds one of these per step and passes it to safeMaterialize INSTEAD of
// the bare AssetIO. After Materialize returns, the executor reads the observed set out
// of the tracker and feeds it into lineage.Writer.CaptureRun for D-04 comparison.
type TrackingIO interface {
    asset.AssetIO          // satisfies the existing user contract; pass-through for Read/Write/PartitionKey
    Observed() []string    // returns the unique upstream names that were Read during materialize (sorted)
}

func NewTrackingIO(inner asset.AssetIO) TrackingIO
```

The executor wires it in `runStep`:
```go
inner := asset.NewAssetIO(a, e, partitionKey)
tracker := asset.NewTrackingIO(inner)
result, runErr := safeMaterialize(stepCtx, a.MaterializeFn(), tracker)  // user code calls tracker.Read; tracker delegates to inner.Read
// ... after materialize ...
observed := tracker.Observed()  // the set of upstream names actually consumed this run
e.deps.LineageWriter.CaptureRun(ctx, tx, runID, a, result, a.CodeHash(), observed)  // observedUpstreams is now a typed param
```

Inside `CaptureRun`:
- Always compute `drift = !sameSet(observedUpstreams, a.Upstreams())`.
- If drift: emit `lineage.drift_detected` event with `{declared, observed, missing, extra}` AND `UPDATE asset_versions SET drift_status='pending' WHERE asset=$1 AND code_hash=$2`.
- Always emit `lineage.captured` carrying both sets (operators have full context regardless of drift state).

**No user opt-in.** Users who never call `io.Read` (e.g., a source asset that scrapes an external API and writes raw rows — no upstream materialization) produce `observed = []`. If their `Upstreams()` is also `[]`, no drift. If their `Upstreams()` is non-empty (declared upstreams that the user code never actually consumed), drift fires correctly — that's exactly the case D-04 wants to catch.

**No `result.Metadata["observed_upstreams"]` shortcut.** Plans 04-04, 04-07 and 04-08 must NOT reference that key. The platform observes upstreams itself via the tracker; user-supplied observed sets are not consulted (would defeat D-04).

This is the architecturally clean realization of D-04 promised by CONTEXT.md: drift detection is the platform's responsibility, not the user's.

**Schema hash dedup (D-08, Pitfall 5):**

package lineage

import (
    "context"
    "database/sql"
    "github.com/google/uuid"
    "github.com/kanpon/data-governance/internal/asset"
    "github.com/kanpon/data-governance/internal/event"
)

type Writer struct {
    db     *sql.DB           // for SyncStaticEdges (uses an internal short tx)
    events event.Writer
}

func NewWriter(db *sql.DB, events event.Writer) *Writer

// SyncStaticEdges UPSERTs asset_edges for every declared upstream of asset (D-01).
// Internal short tx; called from asset.DefinitionRegistry.Register hook.
//
// For each upstream u in asset.Upstreams():
//   INSERT INTO asset_edges (..., from_asset=u, to_asset=asset.Name(), code_hash_first=codeHash, code_hash_latest=codeHash, first_seen_run_id=NULL, ...)
//   ON CONFLICT (from_asset, to_asset) DO UPDATE
//     SET code_hash_latest=EXCLUDED.code_hash_latest, last_seen_at=NOW()
//     WHERE asset_edges.superseded_at IS NULL
//
// Edges previously present but missing from current upstreams: UPDATE superseded_at=NOW() (D-15).
//
// NOTE: Phase 4 first_seen_run_id default is the zero UUID (we'll backfill on first
// run via CaptureRun). Schema design: UUID NOT NULL on asset_edges; use uuid.Nil sentinel.
//
// Returns nil if no upstreams are declared (no-op).
func (w *Writer) SyncStaticEdges(ctx context.Context, a *asset.Asset, codeHash string) error

// CaptureRun is the run-attributed lineage capture hook (D-01 audit trail + D-02 column lineage + D-04 platform-driven drift).
// Called by executor.runStep INSIDE the run-update transaction so atomicity is guaranteed (D-21).
//
// observedUpstreams is the canonical set of upstream names the user's MaterializeFunc
// actually called io.Read() on during materialize. The executor produces it via the
// trackingIO decorator (internal/asset/io_tracking.go) — never derived from
// result.Metadata. Empty slice (len == 0) is a valid value: it means the asset's
// Materialize did not Read any upstream this run.
//
// Steps:
//  1. Resolve column lineage: result.ColumnLineage if non-nil (runtime override D-02);
//     else asset.ColumnLineage() (builder default); else nil → emit metadata
//     {"tag": "column_lineage_undeclared"} on the lineage.captured event.
//  2. UPSERT column_edges rows for each (output, source) pair (D-15 soft-retire pattern).
//  3. UPDATE asset_edges last_seen_* for the upstreams of this asset (run-attribution).
//  4. UPSERT asset_versions row for (asset, codeHash). If new row: INSERT with declared
//     description/owner/tags/columnLineage and drift_status='clean'. If existing row: no-op.
//  5. D-04 PLATFORM-DRIVEN drift detection (always runs, no user opt-in):
//     - drift := !sameSet(observedUpstreams, a.Upstreams())
//     - If drift: emit lineage.drift_detected event with {declared, observed, missing, extra}
//       AND UPDATE asset_versions SET drift_status='pending' WHERE asset=$1 AND code_hash=$2.
//     - If no drift: skip drift-specific writes; asset_versions.drift_status stays 'clean'.
//  6. Emit lineage.captured event with payload
//     {asset, code_hash, observed_upstreams, declared_upstreams, column_lineage_source}.
func (w *Writer) CaptureRun(ctx context.Context, tx *sql.Tx, runID uuid.UUID,
    a *asset.Asset, result asset.MaterializeResult, codeHash string,
    observedUpstreams []string) error
```

}

func NewWriter(events event.Writer) *Writer

// Capture is the schema capture hook (D-05 + D-06 + D-08). Called by executor.runStep
// INSIDE the run-update transaction.
//
// Resolution order:
//  1. conn.(connector.SchemaDescriber).DescribeSchema(ctx, ref) — if available + no error.
//  2. result.Schema — if user's Materialize populated it (D-06 fallback).
//  3. Otherwise: emit lineage.captured event with {"schema_capture": "unsupported"}; return nil.
//
// On error from DescribeSchema (non-fatal per D-08):
//   Emit schema.capture_failed event with {error}; return nil (run continues).
//
// On successful schema capture:
//   1. Compute schemaHash via HashSchema (Pitfall 5: alphabetical column sort).
//   2. SELECT schema_hash, id FROM schema_versions WHERE asset=$1 ORDER BY captured_at DESC LIMIT 1.
//   3a. If hit AND hash matches: UPDATE last_seen_run_id, last_seen_at;
//       emit schema.unchanged event with {asset, schema_hash, version_id}.

//       emit schema.change_detected event with {asset, prev_version_id (nil if no prior), new_version_id, schema_hash}.
//       (Wave 5 plan 04-05 reads this event and writes schema_changes diff rows.)
//   4. Emit schema.captured event with {asset, schema_hash, version_id, capture_source: "descriptor"|"materialize_result"}.
func (w *Writer) Capture(ctx context.Context, tx *sql.Tx, runID uuid.UUID,
    a *asset.Asset, result asset.MaterializeResult,
    conn connector.Connector, ref connector.AssetRef, codeHash string) error

    db := e.deps.Store.DB()

    if err != nil { return fmt.Errorf("executor: begin success tx: %w", err) }
    rollback := true
    defer func() { if rollback { _ = tx.Rollback() } }()

    if e.deps.LineageWriter != nil {

            return fmt.Errorf("executor: lineage capture: %w", err)
        }
    }
    if e.deps.SchemaWriter != nil {
        conn, ref, _ := e.Resolve(a.Name())
        if err := e.deps.SchemaWriter.Capture(ctx, tx, runID, a, result, conn, ref, a.CodeHash()); err != nil {
            // Already non-fatal inside Capture (emits schema.capture_failed); re-raise only on hard tx error.
            return fmt.Errorf("executor: schema capture: %w", err)
        }
    }

    // run-state update INSIDE the tx
    const updRun = `UPDATE runs SET state='succeeded', finished_at=NOW() WHERE id=$1 AND state='running'`
    if _, err := tx.ExecContext(ctx, updRun, runID); err != nil {
        return fmt.Errorf("executor: update run state: %w", err)
    }

    if err := tx.Commit(); err != nil {
        return fmt.Errorf("executor: commit success tx: %w", err)
    }
    rollback = false

    // Step success event AFTER commit (so we don't emit on rolled-back tx):
    e.appendEvent(ctx, runID, event.EventTypeRunStepSucceeded, ...)
    return nil
}

       package asset

       import (
           "context"
           "sort"
           "sync"

           "github.com/kanpon/data-governance/internal/connector"
       )

       // TrackingIO wraps an AssetIO and records every Read(upstream) call so the
       // executor can compare observed-vs-declared upstream sets per D-04.
       //
       // The user's MaterializeFunc sees a normal AssetIO (TrackingIO satisfies the
       // interface). After Materialize returns, the executor calls Observed() to
       // retrieve the unique upstream names actually consumed this run. The lineage
       // writer compares the result with asset.Upstreams() to decide whether to
       // emit lineage.drift_detected and set asset_versions.drift_status='pending'.
       //
       // No user opt-in: every executor-supplied AssetIO is wrapped (Phase 4
       // executor.runStep change in plan 04-04 task 3).
       type TrackingIO interface {
           AssetIO
           Observed() []string
       }

       // NewTrackingIO wraps inner with read-tracking. inner MUST be non-nil.
       func NewTrackingIO(inner AssetIO) TrackingIO {
           return &trackingIO{inner: inner, seen: map[string]struct{}{}}
       }

       type trackingIO struct {
           inner AssetIO
           mu    sync.Mutex
           seen  map[string]struct{}
       }

       func (t *trackingIO) Read(ctx context.Context, upstream string) ([]connector.Row, error) {
           // Record the attempt unconditionally — drift cares about intent, not success.
           t.mu.Lock()
           t.seen[upstream] = struct{}{}
           t.mu.Unlock()
           return t.inner.Read(ctx, upstream)
       }

       func (t *trackingIO) Write(ctx context.Context, rows []connector.Row) (int64, error) {
           return t.inner.Write(ctx, rows)
       }

       func (t *trackingIO) PartitionKey() string { return t.inner.PartitionKey() }

       // Observed returns the unique upstream names that Read() was called with,
       // sorted alphabetically for deterministic comparison. Returns an empty
       // (non-nil) slice if Read was never called.
       func (t *trackingIO) Observed() []string {
           t.mu.Lock()
           defer t.mu.Unlock()
           out := make([]string, 0, len(t.seen))
           for k := range t.seen {
               out = append(out, k)
           }
           sort.Strings(out)
           return out
       }
       ```

       package schema

       import (
           "crypto/sha256"
           "encoding/json"
           "fmt"
           "sort"

           "github.com/kanpon/data-governance/internal/connector"
       )

       // HashSchema returns a stable SHA-256 hex of the canonical JSON of s (D-08 + Pitfall 5).
       //
       // Excluded from the hash by design:
       //   - RowCountEstim (volatile; row count changes constantly without schema change)
       //   - CapturedAt (always different)
       //
       // Columns are sorted ALPHABETICALLY BY NAME before hashing — the PostgreSQL
       // information_schema.columns order is ordinal_position, which can drift if a
       // user does ALTER TABLE DROP COLUMN; ALTER TABLE ADD COLUMN. Alphabetical
       // sort canonicalizes (Pitfall 5).
       //
       // PrimaryKey list IS preserved in original order — composite PK column order
       // is meaningful (PK on (a, b) is different from PK on (b, a)).
       func HashSchema(s connector.Schema) string {
           type canonicalCol struct {
               Name         string  `json:"name"`
               Type         string  `json:"type"`
               Nullable     bool    `json:"nullable"`
               Default      *string `json:"default,omitempty"`
               IsPrimaryKey bool    `json:"is_pk"`
               // Comment intentionally OMITTED — comment changes are non-breaking
               // (D-09) and should not invalidate the schema_hash. Wave 5's diff
               // classifier reads the schema_data JSONB to detect comment changes.
           }
           cols := make([]canonicalCol, 0, len(s.Columns))
           for _, c := range s.Columns {
               cols = append(cols, canonicalCol{
                   Name: c.Name, Type: c.Type, Nullable: c.Nullable,
                   Default: c.Default, IsPrimaryKey: c.IsPrimaryKey,
               })
           }
           sort.Slice(cols, func(i, j int) bool { return cols[i].Name < cols[j].Name })

           type canonicalSchema struct {
               Columns    []canonicalCol `json:"columns"`
               PrimaryKey []string       `json:"pk"`
           }
           pkCopy := append([]string(nil), s.PrimaryKey...) // preserve order
           cs := canonicalSchema{Columns: cols, PrimaryKey: pkCopy}

           b, err := json.Marshal(cs)
           if err != nil {
               panic(fmt.Sprintf("schema: hash marshal: %v", err))
           }
           h := sha256.Sum256(b)
           return fmt.Sprintf("%x", h)
       }

                   },
               })
               return nil
           }

           schemaHash := HashSchema(captured)

           const latestQuery = `
               SELECT id, schema_hash FROM schema_versions
                WHERE asset = $1 ORDER BY captured_at DESC LIMIT 1
           `
           var prevID uuid.UUID
           var prevHash string
           err := tx.QueryRowContext(ctx, latestQuery, a.Name()).Scan(&prevID, &prevHash)
           if err != nil && !errors.Is(err, sql.ErrNoRows) {
               return fmt.Errorf("schema: latest version query: %w", err)
           }
           hasPrev := !errors.Is(err, sql.ErrNoRows)

           now := time.Now().UTC()

           if hasPrev && prevHash == schemaHash {
               // Step 3a: dedup — UPDATE last_seen_*.
               const upd = `UPDATE schema_versions SET last_seen_run_id=$1, last_seen_at=$2 WHERE id=$3`
               if _, err := tx.ExecContext(ctx, upd, runID, now, prevID); err != nil {
                   return fmt.Errorf("schema: dedup update: %w", err)
               }
               _ = w.events.Append(ctx, event.Event{
                   Type: event.EventTypeSchemaUnchanged,
                   ResourceType: "run", ResourceID: runID.String(),
                   Payload: map[string]any{
                       "asset": a.Name(), "schema_hash": schemaHash,
                       "version_id": prevID.String(),
                   },
               })
           } else {
               // Step 3b: INSERT new schema_versions row.
               schemaJSON, _ := json.Marshal(captured)
               newID := uuid.New()
               const ins = `
                   INSERT INTO schema_versions
                       (id, asset, code_hash, schema_hash, schema_data,
                        captured_at, last_seen_at, last_seen_run_id)
                   VALUES ($1, $2, $3, $4, $5::jsonb, $6, $6, $7)
               `
               if _, err := tx.ExecContext(ctx, ins,
                   newID, a.Name(), codeHash, schemaHash, schemaJSON, now, runID); err != nil {
                   return fmt.Errorf("schema: insert version: %w", err)
               }
               // Emit schema.change_detected; payload pointers to the new and prev version.
               // Wave 5 plan 04-05 reads this event (or the new schema_versions row) and writes schema_changes diff rows.
               payload := map[string]any{
                   "asset": a.Name(), "schema_hash": schemaHash,
                   "new_version_id": newID.String(), "code_hash": codeHash,
               }
               if hasPrev {
                   payload["prev_version_id"] = prevID.String()
               }
               _ = w.events.Append(ctx, event.Event{
                   Type: event.EventTypeSchemaChangeDetected,
                   ResourceType: "run", ResourceID: runID.String(),
                   Payload: payload,
               })
           }

           // Step 4: Emit lineage.captured (schema variant) for every successful capture.
           _ = w.events.Append(ctx, event.Event{
               Type: event.EventTypeSchemaCaptured,
               ResourceType: "run", ResourceID: runID.String(),
               Payload: map[string]any{
                   "asset": a.Name(), "schema_hash": schemaHash,
                   "capture_source": captureSource,
               },
           })
           return nil
       }
