# Phase 05 Plan 02: 列策略与仓库原生掩码

**三层列策略表达（builder DSL + REST 运行时 + YAML 标签默认值）配合 COALESCE 优先级，加上 Snowflake DDM 和 BigQuery CLS 仓库原生掩码配置器、异步同步工作器（记录 masking.sync_failed 于永久失败时）以及 15 分钟协调器检测漂移并重新入队收敛。**

## 依赖关系

- 依赖 Phase 05-01（审计哈希链 + Casbin + asset_versions.governance_state 列）
- 提供 RBAC-03（列策略 DSL + Store + REST）和 RBAC-04（Snowflake DDM + BigQuery CLS 同步）
- 影响 Phase 05-03（in-pipeline 掩码）、Phase 05-04（治理审批工作流）、Phase 05-05（通知子系统）

### Task 2: MaskingProvisioner — Snowflake DDM + BigQuery CLS 实现 + River 同步工作器 + reconciler 子命令

**行为：**
- Snowflake `ApplyMaskingPolicy` — 生成完全限定 DDL 模板（`CREATE OR REPLACE MASKING POLICY "DB"."SCH"."dgp_mask_<table>_<column>" ...`），Body 根据 MaskType 切换
- Snowflake `RemoveMaskingPolicy` — `ALTER TABLE ... ALTER COLUMN ... UNSET MASKING POLICY` 然后 `DROP MASKING POLICY ...`
- Snowflake `ListMaskingPolicies` — 查询 `INFORMATION_SCHEMA.MASKING_POLICIES`，按名称前缀解析
- BigQuery `ApplyMaskingPolicy` — 四步流程：确保 Taxonomy（含 FINE_GRAINED_ACCESS_CONTROL）→ 确保 PolicyTag → 设置 IAM → Tables.update
- BigQuery `RemoveMaskingPolicy` — 取消列上附加的 tag
- BigQuery `ListMaskingPolicies` — 读取 BigQuery 表元数据并解析 tag → MaskType
- `PolicySyncWorker` — 消费 `PolicySyncArgs{Asset, Column}` 任务：解析连接器 → 类型断言 → 调用 ApplyMaskingPolicy → 成功则更新 enforcement_mode='warehouse-native' → 永久失败写 `masking.sync_failed` audit + 标记 sync_status='failed'
- `Reconciler.Tick` — 每 15 分钟遍历所有有活动策略的资产，调用 ListMaskingPolicies 比对，差异时写 `masking.sync_drift_detected` 并重新入队，5 分钟宽限期跳过最近更新

## STRIDE 威胁缓解

| Threat ID | 缓解 | 证据 |
|---|---|---|
| T-05-02-01（column_policies 篡改） | mitigate | platform_app 仅 SELECT/INSERT/UPDATE；REST PATCH 由 RequirePermission 保护；每次 Patch 在同一 tx 写 policy.changed |
| T-05-02-02（同步缺失导致泄露） | mitigate | Patch 在同一 tx 入队 River 任务；Reconciler 15 分钟兜底扫描；enforcement_mode 字段记录实际路径 |
| T-05-02-03（Snowflake schema 不匹配） | mitigate | splitTriIdentifier 拒绝非 DB.SCHEMA.TABLE 输入；DDL 使用完全限定标识符 |
| T-05-02-04（BigQuery IAM 传播） | accept | Reconciler GracePeriod = 5min 跳过最近更新 |
| T-05-02-05/06（仓库 SA 泄露） | mitigate（文档） | user_setup 记录最小 IAM；启动 config 不打印 secrets |
| T-05-02-07（River 失败风暴） | mitigate | MaxSyncAttempts=3；永久失败写 audit 并标记 failed |
| T-05-02-08（YAML 篡改） | mitigate | ApplyYAML 每次写入 policy.changed audit 链 |
| T-05-02-10（抵赖） | mitigate | Patch 需要非空 Reason；audit_log 含 actor + before + after + reason |

## 偏差

1. **JSONB allow_roles 而非 TEXT[]** — 项目不依赖 lib/pq，pgx 标准路径处理 TEXT[] 需要额外仪式；切换到 JSONB 与 Phase 4 asset_metadata.tags 模式一致
2. **迁移文件名 20260510000002** — 20260510000000 与既有基线冲突；20260510000001 被 plan 05-01 使用；20260510000003 预留给 plan 05-05
3. **sync_status 列超出计划范围** — PolicySyncWorker 需要向操作员暴露 syncing/synced/failed 状态
4. **River 运行时尚未引入** — Worker 使用抽象接口；生产布线和 cmd/platform/reconciler.go 使用 noopReEnqueuer 占位符
5. **Reconciler v1 ConnectorResolver 返回"未连接"错误** — 连接器解析链由 start 子命令拥有；Reconciler 仍然发出 drift 事件
