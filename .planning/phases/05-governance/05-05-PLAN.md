# Phase 5 Plan 05-05: 数据质量 + Freshness SLA + 通知管道

**目标：在每次成功物化后同事务评估全部质量规则；超过阈值则把 run 标记为质量失败；通过 webhook 或 SMTP 派发告警。**

**Pitfall #6 + Phase 4 D-04 原则：外部 IO（webhook、SMTP）永远异步化；不阻塞 executor / scheduler / governance hot path。**

- 提供 QUAL-01、QUAL-02、QUAL-03、QUAL-04、QUAL-05

### Task 1: QualityRule DSL + connector.QueryAggregate + Postgres 实现 + quality_results 迁移 + evaluator + executor 钩子

**行为：**
- **Migration** 添加：`quality_rules` 表（每 asset/code_hash/rule_name 一行）、`quality_results` 表（每 run/rule 一行）、`runs.run_quality_status` 列、`schedules.last_succeeded_at / freshness_max_lag_seconds / freshness_breach_emitted_at` 列
- **`asset.QualityRule` 接口**：`Name() / Type() / Evaluate(ctx, eval) (QualityResult, error) / ConfigJSON()`
- **NullCheck**：运行 `SELECT COUNT(*)::float, COUNT(*) FILTER (WHERE "<col>" IS NULL)::float FROM ${asset}`；计算 rate，若 rate > MaxRate → failed
- **RangeCheck**：运行 `SELECT MIN("<col>")::float, MAX("<col>")::float FROM ${asset}`；若 MIN < Min 或 MAX > Max → failed
- **SQLAssertion**：替换 `${asset}`，通过 QueryAggregate 运行 SQL；谓词评估标量
- **QualityRule definitions ARE in code_hash** — 规则变更触发新的 asset_version
- **`Evaluator.Evaluate`**：在同一个 executor tx 中运行所有规则；非 QueryAggregate 连接器 → status='error'；每规则有 context.WithTimeout（默认 30s）
- **executor.commitSuccess 钩子**：在 SchemaWriter.Capture 后、tx.Commit 前调用 Evaluator.Evaluate

## STRIDE 威胁缓解

| Threat ID | 缓解 | 证据 |
|---|---|---|
| T-05-05-01（webhook 重放攻击） | mitigate | X-Platform-Timestamp 头 + HMAC 签名；receiver 拒绝 5 分钟之外的请求 |
| T-05-05-02（HMAC 时序攻击） | mitigate | 接收方使用 crypto/subtle.ConstantTimeCompare |
| T-05-05-04（SMTP 凭据泄漏） | mitigate | 通过 env vars 加载；slog 不打印；TLSMandatory |
| T-05-05-05（SMTP 拦截） | mitigate | mail.WithTLSPolicy(mail.TLSMandatory) — 不允许 plaintext fallback |
| T-05-05-06（webhook 风暴） | mitigate | River MaxAttempts=5 + UniqueOpts ByPeriod=1min |
| T-05-05-07（SQL injection in assertion） | accept | SQL 由工程师声明；无运行时用户输入 |
| T-05-05-08（评估超时阻塞 executor tx） | mitigate | context.WithTimeout 包装 QueryAggregate 调用（默认 30s）|
| T-05-05-10（"我没收到告警"抵赖） | mitigate | notification.dispatched / notification.dispatch_failed 进入 event_log |
