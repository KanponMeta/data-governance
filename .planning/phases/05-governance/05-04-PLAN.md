### Task 2: Workflow 服务 + REST + CLI + 物化门 + SLA 扫描仪集成

**行为：**
- **Submit**：验证 code_hash 匹配最新 asset_versions 行；解析审核人池；运行 AutoApprovalChecker.Check。DecisionAutoApproved → 直接转 active；DecisionMustHumanReview/Blocked → 转 in_review + 通知审核人池。
- **Approve/Reject**：需要权限 `/governance/reviews/*` 写权限。Reject 必须有非空 comment（400）。Quorum 逻辑：v1 简化，每个 Approve 追加 `[approved by <uuid>]` 到 comment 字段；只有达到 quorum 数量才翻转状态。
- **Reassign**：轮换 in-flight review 的 reviewer_pool_snapshot；写 event_log（NOT audit_log）；下一决策审计捕获轮换。
- **物化门** (D-08)：在 `executor.runStep`，若 `GovernanceGatingEnabled=true`，在 Resolve+Materialize 之前检查 `asset_versions.governance_state='active'`。默认 **false**（D-08）。
- **SLA 扫描仪**：每调度 tick 运行，检测 `decided_at IS NULL AND submitted_at + sla_hours < NOW()` → 发出 `governance.review_sla_breached` + 通知审核人池和所有者；设置 `sla_breach_emitted_at` 防止重复。

**操作：**
1. 创建 `internal/governance/workflow.go`（Workflow + Submit/Approve/Reject/Reassign/Status）

3. 创建 `internal/governance/sla_scanner.go`（SLAScanner + Scan）
4. 创建 `internal/governance/sla_scanner_test.go`（4 个集成测试）
