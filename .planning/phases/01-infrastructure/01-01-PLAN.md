<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → Container | Docker Compose isolates services; postgres only reachable via compose network |
| Repo → Image | Dockerfile build context defines what code/secrets are copied into the image |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Repository / Docker image | mitigate | `.gitignore` excludes `.env`, `*.key`, build artifacts; `.dockerignore` excludes `.git`, `.env*` so secrets never enter images; `.env.example` ships only placeholder names |
| T-01-02 | Tampering | Build pipeline | mitigate | Dockerfile uses pinned `golang:1.22.x-alpine` base; multi-stage build copies only the binary and CA certs into a `gcr.io/distroless/static-debian12` final image to remove shell/package-manager attack surface |
| T-01-03 | Denial of Service | postgres container | accept | Local dev only; production deployment uses Kubernetes (Phase 2+). Healthchecks fail fast so platform doesn't accept traffic against a dead DB |
| T-01-04 | Elevation of Privilege | Container runtime | mitigate | Final image runs as non-root user (`USER nonroot:nonroot` from distroless); compose sets `read_only: true` where possible |

**ASVS L1 Controls Applied:** V14.1 (Build), V14.2 (Dependency), V14.3 (Unintended Security Disclosure)
</threat_model>

Create `Dockerfile` as a multi-stage build:

```dockerfile
# syntax=docker/dockerfile:1.7

FROM golang:1.22-alpine AS build
WORKDIR /src
RUN apk add --no-cache git ca-certificates
COPY go.mod go.sum ./
RUN go mod download
COPY . .
ARG VERSION=0.1.0-phase1
RUN CGO_ENABLED=0 GOOS=linux go build \
    -trimpath \
    -ldflags "-s -w -X main.version=${VERSION}" \
    -o /out/platform ./cmd/platform

FROM gcr.io/distroless/static-debian12:nonroot AS runtime
WORKDIR /app
COPY --from=build /out/platform /app/platform
USER nonroot:nonroot
EXPOSE 8080
ENTRYPOINT ["/app/platform"]
```

    image: postgres:16-alpine
    environment:
      POSTGRES_USER: platform
      POSTGRES_PASSWORD: platform
      POSTGRES_DB: platform
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

      interval: 5s
      timeout: 5s
      retries: 10

    build:
      context: .
      dockerfile: Dockerfile
    depends_on:

        condition: service_healthy
