## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Connector ABI | mitigate | Connector.APIVersion() returns the constant connector.APIVersion; Registry rejects mismatched versions with ErrIncompatibleVersion; proto service has explicit `package data_governance.connector.v1` so future v2 lives alongside v1 |
| T-04-02 | Tampering | Migration history | mitigate | CI runs `atlas migrate lint --env ci --latest 1` to flag destructive changes; migrations/atlas.sum committed |
| T-04-03 | Information Disclosure | Connector secrets | mitigate | Connector configs travel as `map<string,string> config` in the proto; documentation requires marking sensitive keys (password, api_key) so future logging middleware can redact them. Phase 2 enforces; Phase 1 documents the convention in proto comments. |
| T-04-04 | Tampering | Proto IDL | mitigate | buf lint runs in CI; buf breaking checks against the previous main branch. Proto file imports only well-known types (no third-party messages) — no transitive trust required. |

| T-04-06 | Repudiation | Connector registration | mitigate | Logging via slog at INFO level for audit traceability; full event emission deferred to Phase 2 (Phase 1 D-10 enum does not include connector lifecycle events) |
